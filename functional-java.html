Introduction to Maven

Maven is a powerful project management and comprehension tool that is used for managing project builds, dependencies, and documentation. It simplifies the build process like ANT, but it also offers a standard structure for projects, clear project definitions, and a host of plugins for various tasks.

Key Concepts

  1. POM (Project Object Model): The fundamental unit of work in Maven is the POM file, an XML file that contains information about the project and configuration details used by Maven to build the project.
  2. Dependencies: Libraries or other projects that your project depends on.
  3. Repositories: Directories where Maven stores and retrieves dependencies. There are local, central, and remote repositories.
  4. Plugins: Provide goals to perform tasks such as compiling code, running tests, creating JAR files, etc.

Installation

Windows

  1. Download Maven from the official website.
  2. Extract the archive and set the MAVEN_HOME environment variable to the extracted directory.
  3. Add the bin directory to the PATH environment variable.

macOS

  1. Install Homebrew if you don't have it: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Install Maven: brew install maven

Linux

  1. Install via package manager: 
    sudo apt-get install maven # Debian/Ubuntu
sudo yum install maven     # Fedora

Basic Commands

  1. Creating a New Maven Project 
    mvn archetype:generate -DgroupId=com.mycompany.app -DartifactId=my-app -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false
  2. Building a Project 
    mvn clean install
  3. Running Tests 
    mvn test
  4. Packaging a Project 
    mvn package
  5. Running a Maven Project 
    mvn exec:java -Dexec.mainClass="com.mycompany.app.App"

Understanding the POM File

Basic Structure

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.mycompany.app</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0-SNAPSHOT</version>
</project>

Adding Dependencies

<dependencies>
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.12</version>
        <scope>test</scope>
    </dependency>
</dependencies>

Adding Plugins

<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.8.1</version>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
            </configuration>
        </plugin>
    </plugins>
</build>

Working with Repositories

  1. Local Repository: Located in the .m2 directory in your home directory.
  2. Central Repository: Maven's default repository, located at repo.maven.apache.org.
  3. Remote Repository: Specify in your settings.xml or pom.xml to use repositories other than the default central repository.

Maven Lifecycle

  1. Default: Handles project deployment.
  2. Clean: Handles project cleaning.
  3. Site: Handles the creation of project documentation.

Default Lifecycle Phases

  1. validate: Validate the project is correct and all necessary information is available.
  2. compile: Compile the source code of the project.
  3. test: Test the compiled source code using a suitable testing framework.
  4. package: Take the compiled code and package it in its distributable format, such as a JAR.
  5. verify: Run any checks to verify the package is valid and meets quality criteria.
  6. install: Install the package into the local repository for use as a dependency in other projects locally.
  7. deploy: Copy the final package to the remote repository for sharing with other developers and projects.

Best Practices

  1. Consistent Directory Layout: Follow the standard Maven directory layout.
  2. Versioning: Use meaningful version numbers for your artifacts.
  3. Dependency Management: Avoid version conflicts by using dependency management.
  4. Profiles: Use profiles to customize builds for different environments.
  5. Documentation: Keep your POM file well-documented with comments.
